# BFI 职业性格测评系统 - 产品需求文档

> 版本：v1.2
> 更新日期：2026-03-27
> 状态：当前版本功能定稿，供前后端对齐开发

---

## 一、产品概述

基于大五人格量表（BFI-44）的职业性格测评工具，结合 AI 深度分析，为用户提供性格画像、职场推荐与成长建议。支持 BFI 标准测评、MBTI/荣格八维手动输入两种数据来源。BFI 测评可选上传简历获取 AI 深度报告；手动输入 MBTI/荣格数据时必须上传简历，结果仅展示 AI 深度报告（不展示大五维度分数）。

简历上传支持多来源：本地文件上传、在线简历导入、在线附件导入，并与在线简历系统联动。

---

## 二、核心流程

```
首页 → 测评主页 → BFI 答题 → 上传简历（可跳过）→ 结果页（测评结果 + 深度报告）
                  → 手动输入 → 上传简历（不可跳过）→ 结果页（仅深度报告）
                  ↘ 历史记录 → 历史详情
简历上传页 → 附件管理页（选择在线附件）
```

## 三、各环节详细需求

### 3.1 测评主页（WELCOME）

**页面元素：**
- 顶部左上角：返回首页按钮
- 顶部右上角：历史记录图标，点击进入历史记录列表
- 中间：标题"职业性格测评"、介绍文案、五维度标签轮播动画
- 功能列表：44 题科学测评 / 五维雷达图 + 常模对比 / AI 深度分析 + 岗位推荐
- 底部主按钮："开始测评"
- 主按钮下方文字链接："已有 MBTI / 荣格八维数据？点此输入 →"

**进度恢复弹窗：**

进入测评主页时，检测 localStorage 中是否有未完成/未提交的测评进度：

| 情况 | 弹窗标题 | 弹窗内容 | 按钮 |
|------|---------|---------|------|
| 答题中途退出（未答完 44 题） | 检测到未完成的测评 | 上次测评未完成，已完成 X/44 题。是否继续上次的进度？ | [继续答题] [重新开始] |
| 答完题但在简历步骤退出 | 检测到已完成的测评 | 上次测评已完成答题，还未上传简历生成报告。是否继续？ | [继续上传简历] [重新开始] |
| 无进度 | 不弹窗 | — | — |

**判断逻辑：**
- 从 localStorage 读取 `bfi_quiz_progress`
- 有数据且 `Object.keys(answers).length < shuffledIds.length` → 答题中途退出
- 有数据且 `Object.keys(answers).length >= shuffledIds.length` → 答完题在简历步骤退出（此时自动计算 BFI 分数，直接恢复到 RESUME 阶段）
- 无数据 → 不弹窗

### 3.2 BFI-44 答题（QUIZ）

**题库：** 44 道 BFI 标准题目，涵盖五个维度：
- E（外向性）：8 题
- A（宜人性）：9 题
- C（尽责性）：9 题
- N（神经质/情绪性）：8 题
- O（开放性）：10 题

**答题规则：**
- 每次测评随机打乱题目顺序
- 逐题展示，5 级李克特量表（1=非常不同意 ~ 5=非常同意）
- 支持回退到上一题
- 部分题目为反向计分（score = 6 - raw）

**计分方式：**
- 各维度均分 = 该维度得分总和 / 该维度题目数
- 分数范围：1.00 ~ 5.00
- 常模参考值：E=3.10, A=3.75, C=3.36, N=3.05, O=3.69

**进度持久化：**
- 每次作答后自动保存到 localStorage（key: `bfi_quiz_progress`）
- 保存内容：`{ shuffledIds, currentIndex, answers, timestamp }`
- 答完最后一题自动计算分数并进入简历步骤，进度同步保存（确保简历步骤退出时 44 题答案完整）
- 测评完成后（提交简历或跳过）清除进度

### 3.3 手动输入（MANUAL_INPUT）

支持用户输入已有的人格测评数据，两种维度可选输入：

**MBTI 输入：**
- 四个维度各提供下拉选择：E/I、S/N、T/F、J/P
- 组合为 4 位字母（如 ENFP）

**荣格八维输入：**
- 8 个认知功能各提供滑块：Ti、Te、Fi、Fe、Ni、Ne、Si、Se
- 范围 0 ~ 100

**数据处理原则：**
- MBTI 与荣格八维属于不同的理论框架，**不做向大五人格（BFI）的跨框架近似映射**，因为这种映射缺乏充分的心理学依据，容易产生误导性结果
- 手动输入的 MBTI/荣格数据直接传递给 AI 用于生成深度报告，不转换为 BFI 分数
- 手动输入模式下，结果页**不展示大五维度分数和雷达图**，仅展示 AI 深度报告
- 手动输入模式下，**不允许跳过简历上传**（因为没有 BFI 分数可展示，跳过后无任何可用结果）

**手动输入进度持久化：**
- 输入过程中自动保存到 localStorage（key: `bfi_manual_progress`）
- 保存内容：`{ mode, mbtiSelections, jungScores, completed, mbtiType, jungScoresResult, timestamp }`
- 提交或重新开始时清除进度

### 3.4 上传简历（RESUME）

**页面布局：**
- 顶部：PageHeader，标题"上传你的简历"
- 副标题："AI将结合你的性格数据与职业经历生成一份专属的深度职场画像"
- 内容区域：根据在线简历数据状态展示不同 UI
- 底部主按钮："生成 AI 分析报告"（有文本时可点击）
- 次要链接："跳过，仅查看测评结果"（仅 BFI 测评模式显示）

**两种显示状态：**

| 状态 | 触发条件 | UI 形态 |
|------|---------|--------|
| picker（来源选择） | 用户有在线简历或在线附件 | 来源选择卡片 + 文本粘贴框 |
| empty（空状态） | 无在线简历且无在线附件 | 文件上传按钮 + 文本粘贴框 |

**picker 状态下的来源选择：**

页面进入时异步加载在线简历和在线附件列表，根据可用数据动态展示来源选项：

| 来源 | 展示条件 | 点击行为 |
|------|---------|---------|
| 在线简历 | 有在线简历数据 | 导入结构化简历为纯文本 |
| 附件导入 | 有在线附件数据 | 跳转至附件管理页选择附件 |
| 文件上传 | 始终显示 | 弹出 ActionSheet 选择上传方式 |

来源选择卡片高亮当前选中来源（绿色背景 + 绿色边框），未选中为灰色底色。

**文件上传 ActionSheet：**

点击"文件上传"按钮后弹出底部 ActionSheet，提供以下选项：

| 选项 | 状态 | 说明 |
|------|------|------|
| 本地文件 | 可用 | 打开文件选择器，支持 TXT、MD、DOC、DOCX、PDF |
| 上传图片 | 可用 | 打开图片选择器 |
| 微信上传 | 禁用（灰色） | 预留功能，当前不可用 |

**PDF 解析：** 前端使用 pdfjs-dist 提取纯文本，无需后端。

**在线简历联动：**
- 上传本地文件时，自动将文件同步到在线附件库（调用 `uploadAttachmentToOnline`）
- 上传完成后触发 `onFileUploaded` 回调，用于标记有新上传
- 返回首页时若检测到有新上传，弹出"同步至在线简历"提示弹窗

**简历草稿状态：**
- 跳转至附件管理页前，当前文本和来源保存为草稿（`draftResumeText`, `draftImportSource`）
- 从附件管理页返回时自动恢复草稿内容
- 从附件管理页选择附件后，自动填入附件信息

**跳过逻辑（仅 BFI 测评模式可跳过）：**
- **手动输入模式（MBTI/荣格）不显示跳过按钮**，必须上传简历才能生成报告
- BFI 测评模式跳过后不生成 AI 深度报告
- 仍然保存测评记录到历史（标记 `resumeSkipped: true`）
- 仍然基于性格数据匹配岗位推荐
- 结果页只显示"测评结果"Tab，不显示"深度报告"Tab
- 结果页底部显示"上传简历，获取 AI 深度报告"入口，可随时补充

**返回逻辑：**

| 来源页面 | 返回行为 |
|---------|---------|
| QUIZ（BFI 答题后） | 弹出退出确认弹窗："确定退出吗？" + "答题进度已保存，下次可继续上传简历。"，按钮 [继续上传] [确定退出] |
| MANUAL_INPUT（手动输入后） | 直接返回手动输入页 |
| RESULT（结果页补充上传） | 直接返回结果页 |
| HISTORY_DETAIL（历史详情补充上传） | 直接返回历史详情页 |

### 3.6 AI 报告生成（LOADING）

**触发条件：** 用户在简历页点击"生成 AI 分析报告"

**并行请求：** 同时发起 2 个 AI 接口调用：

| 接口 | 返回格式 | 用途 |
|------|---------|------|
| 深度报告生成 | Markdown 流式 | 四大板块报告内容（含成长引擎三子模块） |
| 岗位类型推荐 | JSON | 2-3 个岗位方向 |

> 注：成长建议（成长引擎）已整合到深度报告 Markdown 中输出，不再作为独立接口调用。报告的"成长引擎"板块包含三个子模块：AI 时代突围、习惯养成、技能训练。

**输入数据（三种路径互斥）：**
- BFI 五维度分数（BFI 测评时提供，手动输入模式无此数据）
- MBTI 类型（MBTI 手动输入时提供）
- 荣格八维分数（荣格手动输入时提供）
- 简历纯文本（始终存在）

**深度报告 Prompt 结构（Markdown 输出）：**

```
## 核心画像

**核心优势**（2-3条）
每条格式：「**要点标题**：描述内容」，要点间空行分隔

**潜在短板**（2-3条）
同上格式

**内在张力**
找到性格中最有趣的矛盾组合，深入剖析

## 最佳战场

**适合的公司类型**
文化氛围、公司阶段、团队规模，具体到可执行

**适合的团队角色**
在团队中应扮演什么位置

**理想搭档画像**
需要什么性格特质的上级和同事来互补

**最适合的岗位类型**
结合简历背景给出 2-3 个具体岗位方向

## 成长引擎

### AI 时代突围
结合性格特征和简历背景，AI 时代定位建议

### 习惯养成
针对最突出的性格短板，给出具体可操作的行为处方

### 技能训练
结合简历背景和推荐岗位，列出最该补的 1-2 个技能

## 寄语
> 引用格式，1-2 句收尾
```

**Prompt 格式规则：**
- 板块标题用 `##`，子模块标题用 `###`
- 子项标题（如 **核心优势**）独占一行
- 禁止列表符号（`* - 1.` 等），多个要点用「**要点标题**：描述内容」格式
- 三种输入路径有不同的分析指引（BFI 关注维度偏离和组合效应，MBTI 关注类型在简历中的具体表现，荣格关注功能对的平衡）

**岗位类型推荐 JSON 结构：**
```json
[
  { "type": "岗位类型名称", "reason": "推荐理由", "tags": ["标签1", "标签2"] }
]
```

**异常处理：**
- API 调用失败时，仍保存历史记录（有分数，无报告）
- 用户可在结果页/历史详情页通过"上传简历"入口重新生成

**API 配置：**
- baseUrl: 环境变量 `VITE_API_BASE_URL`（默认 OpenAI）
- apiKey: 环境变量 `VITE_API_KEY`
- model: 环境变量 `VITE_MODEL`（默认 gpt-4o-mini）
- 无 API Key 时使用前端 Mock 数据（根据 BFI 分数/MBTI/荣格类型生成结构化的模拟报告）

### 3.7 测评结果（RESULT）

**Tab 结构：**

| 条件 | 显示的 Tab | 默认 Tab |
|------|-----------|----------|
| BFI 测评 + 有 AI 报告 | [测评结果] [深度报告] | 测评结果 |
| BFI 测评 + 无 AI 报告（跳过简历或 API 失败） | [测评结果] | 测评结果 |
| 手动输入（MBTI/荣格） | [深度报告] | 深度报告 |

> **手动输入模式不展示"测评结果"Tab**：因为 MBTI/荣格数据不做向 BFI 的跨框架映射，没有大五维度分数可供展示。手动输入模式下若报告生成失败，显示"报告生成失败"提示和重新上传入口。

#### Tab 1：测评结果

**五维人格雷达图：**
- 用户得分 vs 常模参考值双线对比
- 用户得分：绿色实线
- 常模参考：灰色虚线

**雷达图下方分数卡片：**
- 5 列，每列：维度得分 + 维度名称 + 与常模差值（+/-）

**各维度得分详情（进度条）：**
- 每个维度一行：维度名称 + 得分
- 进度条范围 1.0 ~ 5.0，主题色填充
- 常模位置用竖线标记

**维度说明卡片：**
- 5 张卡片，每张含维度名称 + 文字说明

**补充上传入口（仅无报告时显示）：**
- 测评结果底部展示卡片按钮
- 文案："上传简历，获取 AI 深度报告"
- 副文案："结合简历生成职场画像、岗位推荐与成长建议"
- 点击后跳转至简历上传页

#### Tab 2：深度报告（仅有报告时显示）

**核心画像板块：**
- 从报告 Markdown 中提取 `## 核心画像` 段落
- 按 `**粗体标题**` 拆分为子卡片（如核心优势、潜在短板、内在张力）
- 每张子卡片：圆形指示点 + 标题 + 内容

**最佳战场板块：**
- 从报告 Markdown 中提取 `## 最佳战场` 段落
- 按 `**粗体标题**：内容` 格式拆分为子卡片
- 每张子卡片：标题 + 内容
- 板块底部：岗位推荐卡片列表 + "换一换"按钮

**成长引擎板块：**
- 从报告 Markdown 中提取 `## 成长引擎` 段落
- 包含三个子模块：`### AI 时代突围`、`### 习惯养成`、`### 技能训练`
- 每个子模块独立展示

**寄语板块：**
- 从报告 Markdown 中提取 `## 寄语` 段落
- 引用块样式展示

**免责声明：**
- 底部黄色提示框
- "本报告由 AI 基于性格测评数据与简历内容生成，仅供参考，不构成专业心理咨询或职业规划建议。"

### 3.8 岗位推荐

**数据来源：** 前端岗位库（35 个岗位），根据性格分数 + 荣格分数 + MBTI 类型匹配

**展示规则：**
- 每次展示 3 个岗位卡片
- "换一换"按钮刷新推荐
- 每次从匹配池中随机抽取，避免重复
- 已展示过的岗位 ID 记录在 `excludedJobIds` 中
- 当所有匹配岗位都展示过后，重置排除列表重新随机
- 点击岗位卡片跳转至岗位详情页

### 3.9 历史记录（HISTORY）

**入口：** 测评主页右上角历史图标

**返回：** 返回首页（HOME）

**列表展示：**
- 按时间倒序排列
- 每条记录显示：
  - 时间戳（格式：YYYY-MM-DD HH:mm）
  - 测评类型标签：BFI 测评（绿色）/ MBTI 输入（紫色）/ 荣格输入（琥珀色）
  - 性格标签（如"ENFP · 高外向性"、"高外向性 · 低神经质"、"Ni-Te"）
  - 报告状态：
    - 有报告 → "已生成报告"
    - 跳过简历 → "仅测评结果" + "未上传简历"灰色标签
  - 右侧："查看详情 →"
  - 右上角：删除按钮

**性格标签生成规则：**
- BFI 测评：取与常模差距最大的高维度和低维度（差值 > 0.2），格式"高X · 低Y"或"高X"或"均衡型"
- MBTI 输入：直接使用 MBTI 类型，如"ENFP"
- 荣格输入：取排名前两的认知功能，格式"Ni-Te"
- 混合展示：各部分用" · "连接

**删除操作：**
- 点击删除按钮弹出确认弹窗
- 标题："删除记录"
- 内容："确定要删除这条测评记录吗？删除后无法恢复。"
- 按钮：[取消] [删除]

**空状态：** 图标 + "暂无测评记录" + "完成一次测评后记录将出现在这里"

**存储限制：** 最多保留最近 20 条记录，超出自动淘汰最早的

---

### 3.10 历史详情（HISTORY_DETAIL）

**页面结构：** 与结果页一致

**Meta 信息：** 时间戳 + 性格标签

**Tab 逻辑：** 同结果页——BFI 测评有报告时显示两个 Tab，无报告时只显示"测评结果"；手动输入模式只显示"深度报告"Tab

**补充上传入口：** 无报告的历史记录，测评结果底部展示"上传简历，获取 AI 深度报告"卡片按钮。点击后将该记录的分数数据加载到状态中，跳转至简历上传页。生成报告后更新该历史记录（而非新建记录）。

**岗位推荐：** 从历史记录的分数数据实时匹配，支持"换一换"刷新

## 四、数据模型

### 4.1 历史记录（History Record）

```
{
  id: string,                    // 唯一标识，格式 "rec_{timestamp}"
  timestamp: number,             // 创建时间戳
  assessmentType: string,        // "bfi" | "mbti" | "jung"
  bfiScores: {                   // BFI 五维度均分（BFI 测评时有值）
    E: number,                   // 1.00 ~ 5.00
    A: number,
    C: number,
    N: number,
    O: number
  } | null,
  mbtiType: string | null,       // 如 "ENFP"（MBTI 手动输入时有值）
  jungScores: {                  // 荣格八维分数（荣格手动输入时有值）
    Ti: number, Te: number,      // 0 ~ 100
    Fi: number, Fe: number,
    Ni: number, Ne: number,
    Si: number, Se: number
  } | null,
  personalityTag: string,        // 如 "高外向性 · 低神经质"、"ENFP"、"Ni-Te"
  resumeText: string,            // 简历纯文本（跳过时为空）
  report: string,                // AI 生成的 Markdown 报告（跳过/失败时为空）
  jobTypeRecs: Array,            // AI 岗位类型推荐 JSON
  resumeSkipped: boolean         // 是否跳过了简历上传（仅跳过时为 true）
}
```

### 4.2 答题进度（Quiz Progress）

```
{
  shuffledIds: number[],         // 打乱后的题目 ID 序列
  currentIndex: number,          // 当前题目索引（0-based）
  answers: { [questionId]: 1~5 },// 已作答的题目及选项
  timestamp: number              // 最后保存时间
}
```

### 4.3 手动输入进度（Manual Progress）

```
{
  mode: string,                  // 输入模式（"mbti" | "jung"）
  mbtiSelections: object,        // MBTI 四维选择状态
  jungScores: object,            // 荣格八维滑块值
  completed: boolean,            // 是否已填写完成
  mbtiType: string | null,       // 组合后的 MBTI 类型
  jungScoresResult: object | null,// 最终荣格分数
  timestamp: number              // 最后保存时间
}
```

### 4.4 简历缓存（Resume Cache）

```
{
  text: string,                  // 简历纯文本
  source: string,                // 来源（文件名或 "manual_input"）
  sourceType: string,            // 来源类型
  timestamp: number              // 缓存时间
}
```

### 4.5 在线简历（Online Resume）

```
{
  name: string,                  // 姓名
  title: string,                 // 职位
  education: string,             // 教育背景
  experience: string,            // 工作经历
  skills: string                 // 技能特长
}
```

### 4.6 在线附件（Online Attachment）

```
{
  id: string,                    // 唯一标识，格式 "att_{timestamp}"
  fileName: string,              // 文件名
  fileType: string,              // 文件类型（"pdf" | "doc" | "other"）
  category: string,              // 分类（如 "简历"、"作品"）
  uploadTime: number             // 上传时间戳
}
```

## 六、状态机流转

```
HOME
  │ 点击"开始测评"
  ▼
WELCOME
  ├─ 点击"开始测评"────────────→ QUIZ
  ├─ 点击"手动输入"────────────→ MANUAL_INPUT（清除手动输入进度）
  ├─ 点击"历史记录"────────────→ HISTORY
  └─ 有进度弹窗 → 继续 ───────→ QUIZ 或 RESUME（取决于答题进度）
                 → 重新开始 ──→ 清除进度，留在 WELCOME

QUIZ
  │ 答完第 44 题，自动计算分数
  ▼
RESUME ←── MANUAL_INPUT 提交后也进入此步骤
  ├─ 提交简历 ─→ LOADING → RESULT
  ├─ 跳过简历 ─→ RESULT（无报告，仅 BFI 模式可跳过，手动输入模式无此选项）
  ├─ 点击"附件导入" ──→ ATTACHMENT_MANAGE
  │                        └─ 选择附件 ──→ RESUME（自动填入附件信息）
  │                        └─ 返回 ──→ RESUME（恢复草稿）
  ├─ 返回（来自 QUIZ）──→ 弹出退出确认弹窗
  │     └─ 确定退出 ──→ WELCOME
  │     └─ 继续上传 ──→ 留在 RESUME
  ├─ 返回（来自 MANUAL_INPUT）──→ MANUAL_INPUT
  ├─ 返回（来自 RESULT）──→ RESULT
  └─ 返回（来自 HISTORY_DETAIL）──→ HISTORY_DETAIL

RESULT
  ├─ 点击返回 ──→ WELCOME
  ├─ 点击岗位 ──→ JOB_DETAIL
  └─ 点击"上传简历"（无报告时）──→ RESUME

HISTORY → 点击记录 → HISTORY_DETAIL
  ├─ 返回 ──→ HOME

HISTORY_DETAIL
  ├─ 点击岗位 ──→ JOB_DETAIL
  ├─ 返回 ──→ HISTORY
  └─ 点击"上传简历"（无报告时）──→ RESUME（加载历史分数，标记补充记录 ID）

JOB_DETAIL
  └─ 返回 ──→ 来源页面（RESULT 或 HISTORY_DETAIL）
```

## 七、历史记录写入时机

| 场景 | 是否写入历史 | 记录内容 |
|------|-------------|---------|
| 提交简历 + AI 生成成功 | 是（新建或更新补充记录） | 完整记录（分数 + 简历 + 报告 + 推荐） |
| 提交简历 + AI 生成失败 | 是（新建或更新补充记录） | 部分记录（分数 + 简历，报告为空） |
| 跳过简历 | 是 | 部分记录（分数，标记 `resumeSkipped: true`） |
| 答题中途退出 | 否 | 进度保存在 `bfi_quiz_progress`，下次可恢复 |
| 简历步骤退出 | 否 | 进度保存在 `bfi_quiz_progress`，下次可恢复至 RESUME |

> 从历史详情页补充上传简历时，生成报告后更新原历史记录（`updateHistoryRecord`），而非新建记录。

## 八、接口需求（后端）

### 8.1 深度报告生成

```
POST /chat/completions
Content-Type: application/json
Authorization: Bearer {API_KEY}

Request:
{
  model: string,
  messages: [
    { role: "system", content: "你是一位精通心理学和职业规划的顶级专家。请用中文回答，使用 Markdown 格式。" },
    { role: "user", content: "{prompt}" }
  ],
  stream: true,
  temperature: 0.8,
  max_tokens: 3000
}

Response: SSE 流式返回 Markdown 文本
```

**Prompt 输入变量（三种路径互斥）：**
- 路径一：BFI 五维度均分 + 常模参考 + 简历纯文本
- 路径二：MBTI 类型 + 简历纯文本
- 路径三：荣格八维认知功能分数（按强度排序）+ 简历纯文本

### 8.2 岗位类型推荐

```
POST /chat/completions

Request:
{
  model: string,
  messages: [
    { role: "system", content: "你是职业规划专家。请严格按JSON格式返回。" },
    { role: "user", content: "{prompt}" }
  ],
  temperature: 0.7,
  max_tokens: 500
}

Response: JSON 数组
[{ "type": string, "reason": string, "tags": string[] }]
```

> 注：当前两个接口均通过 OpenAI Chat Completions 格式调用。如后端需自建网关代理，保持该接口格式即可无缝对接。

- 全局入场动画：`fadeInUp`（上移淡入）、`fadeIn`（淡入）、`slideInRight`（右滑入）
- 安全区适配：顶部 `safe-top`、底部 `safe-bottom`
