# BFI 职业性格测评系统 - 产品需求文档

> 版本：v1.0
> 更新日期：2026-03-03
> 状态：当前版本功能定稿，供前后端对齐开发

---

## 一、产品概述

基于大五人格量表（BFI-44）的职业性格测评工具，结合 AI 深度分析，为用户提供性格画像、职场推荐与成长建议。支持 BFI 标准测评、MBTI/荣格八维手动输入两种数据来源，可选上传简历获取 AI 深度报告。

---

## 二、核心流程

```
首页 → 测评主页 → [BFI 答题 / 手动输入] → 上传简历（可跳过）→ 结果页
                  ↘ 历史记录 → 历史详情
```

### 2.1 页面清单

| 页面 | 路径标识 | 说明 |
|------|---------|------|
| 首页 | HOME | App 入口，导航至测评 |
| 测评主页 | WELCOME | 开始测评 / 手动输入 / 历史记录入口 |
| BFI 答题 | QUIZ | 44 道题逐题作答 |
| 手动输入 | MANUAL_INPUT | MBTI 四维选择 + 荣格八维滑块 |
| 上传简历 | RESUME | 文件上传 / 文本粘贴 / 跳过 |
| 加载中 | LOADING | AI 报告生成动画 |
| 测评结果 | RESULT | 测评结果 + 深度报告（两个 Tab） |
| 历史记录 | HISTORY | 所有测评记录列表 |
| 历史详情 | HISTORY_DETAIL | 单条历史记录的完整结果 |
| 岗位详情 | JOB_DETAIL | 推荐岗位的详细信息 |

## 三、各环节详细需求

### 3.1 测评主页（WELCOME）

**页面元素：**
- 顶部左上角：返回首页按钮
- 顶部右上角：历史记录图标，点击进入历史记录列表
- 中间：标题"职业性格测评"、介绍文案、五维度标签轮播动画
- 功能列表：44 题科学测评 / 五维雷达图 + 常模对比 / AI 深度分析 + 岗位推荐
- 底部主按钮："开始测评"
- 主按钮下方文字链接："已有 MBTI / 荣格八维数据？点此输入 →"

**进度恢复弹窗：**

进入测评主页时，检测 localStorage 中是否有未完成/未提交的测评进度：

| 情况 | 弹窗标题 | 弹窗内容 | 按钮 |
|------|---------|---------|------|
| 答题中途退出（未答完 44 题） | 检测到未完成的测评 | 上次测评未完成，已完成 X/44 题。是否继续上次的进度？ | [继续答题] [重新开始] |
| 答完题但在简历步骤退出 | 检测到已完成的测评 | 上次测评已完成答题，还未上传简历生成报告。是否继续？ | [继续上传简历] [重新开始] |
| 无进度 | 不弹窗 | — | — |

**判断逻辑：**
- 从 localStorage 读取 `bfi_quiz_progress`
- 有数据且 `Object.keys(answers).length < shuffledIds.length` → 答题中途退出
- 有数据且 `Object.keys(answers).length >= shuffledIds.length` → 答完题在简历步骤退出
- 无数据 → 不弹窗

---

### 3.2 BFI-44 答题（QUIZ）

**题库：** 44 道 BFI 标准题目，涵盖五个维度：
- E（外向性）：8 题
- A（宜人性）：9 题
- C（尽责性）：9 题
- N（神经质/情绪性）：8 题
- O（开放性）：10 题

**答题规则：**
- 每次测评随机打乱题目顺序
- 逐题展示，5 级李克特量表（1=非常不同意 ~ 5=非常同意）
- 支持回退到上一题
- 部分题目为反向计分（score = 6 - raw）

**计分方式：**
- 各维度均分 = 该维度得分总和 / 该维度题目数
- 分数范围：1.00 ~ 5.00
- 常模参考值：E=3.10, A=3.75, C=3.36, N=3.05, O=3.69

**进度持久化：**
- 每次作答后自动保存到 localStorage（key: `bfi_quiz_progress`）
- 保存内容：`{ shuffledIds, currentIndex, answers, timestamp }`
- 答完最后一题及进入简历步骤后也保存（确保简历步骤退出时 44 题答案完整）
- 测评完成后（提交简历或跳过）清除进度

---

### 3.3 手动输入（MANUAL_INPUT）

支持用户输入已有的人格测评数据，两种维度可同时输入：

**MBTI 输入：**
- 四个维度各提供下拉选择：E/I、S/N、T/F、J/P
- 组合为 4 位字母（如 ENFP）

**荣格八维输入：**
- 8 个认知功能各提供滑块：Ti、Te、Fi、Fe、Ni、Ne、Si、Se
- 范围 0 ~ 100

**数据转换：**
- MBTI → BFI 近似分（基于心理学文献映射）
- 荣格八维 → BFI 近似分（基于功能特征推导）
- BFI 实测数据优先级最高；若无 BFI，取荣格+MBTI 均值；若只有一种则用该种

---

### 3.4 上传简历（RESUME）

**页面布局：**
- 标题："上传你的简历"
- 副标题："AI 将结合你的性格数据与职业经历，生成一份专属的深度职场画像"
- 文件上传区域（支持点击选择 + 拖拽）
- 分割线"或直接粘贴"
- 文本输入框
- 主按钮："生成 AI 分析报告"（有文本时可点击）
- 次要链接："跳过，仅查看测评结果"

**支持的文件格式：** TXT、MD、DOC、DOCX、PDF

**PDF 解析：** 前端使用 pdfjs-dist 提取纯文本，无需后端。

**简历缓存机制：**
- 提交简历时自动缓存到 localStorage（key: `bfi_resume_cache`）
- 缓存内容：`{ text, source, timestamp }`
- 下次进入简历页时，**自动填入**缓存的文本和文件名，无弹窗
- 已有缓存时上传区域显示文件名 + "点击重新上传"提示
- 点击上传区域即可重新选择文件覆盖
- 跳过时不写入缓存

**跳过逻辑：**
- 跳过后不生成 AI 深度报告
- 仍然保存测评记录到历史（标记 `resumeSkipped: true`）
- 仍然基于性格数据匹配岗位推荐
- 结果页只显示"测评结果"Tab，不显示"深度报告"Tab
- 结果页底部显示"上传简历，获取 AI 深度报告"入口，可随时补充

---

### 3.5 AI 报告生成（LOADING）

**触发条件：** 用户在简历页点击"生成 AI 分析报告"

**并行请求：** 同时发起 3 个 AI 接口调用：

| 接口 | 返回格式 | 用途 |
|------|---------|------|
| 深度报告生成 | Markdown 流式 | 四大板块报告内容 |
| 岗位类型推荐 | JSON | 2-3 个岗位方向 |
| 成长建议 | JSON | 3 条成长建议 |

**输入数据：**
- BFI 五维度分数（或转换后的近似分）
- MBTI 类型（如有）
- 荣格八维分数（如有）
- 简历纯文本

**深度报告 Prompt 结构（Markdown 输出）：**

```
## 核心画像
核心优势（2-3条）、潜在短板（2-3条）、内在张力或和谐点

## 最佳战场
- **适合的公司类型**
- **适合的团队角色**
- **理想搭档画像**
- **最适合的岗位类型**（2-3 个方向及原因）

## 成长引擎
3 条成长建议，每条含标题 + 具体描述

## 寄语
1-2 句激励性总结
```

**岗位类型推荐 JSON 结构：**
```json
[
  { "type": "岗位类型名称", "reason": "推荐理由", "tags": ["标签1", "标签2"] }
]
```

**成长建议 JSON 结构：**
```json
[
  { "title": "建议标题", "description": "具体建议", "tags": ["标签1", "标签2"] }
]
```

**异常处理：**
- API 调用失败时，仍保存历史记录（有分数，无报告）
- 用户可在结果页/历史详情页通过"上传简历"入口重新生成

**API 配置：**
- baseUrl: 环境变量 `VITE_API_BASE_URL`（默认 OpenAI）
- apiKey: 环境变量 `VITE_API_KEY`
- model: 环境变量 `VITE_MODEL`（默认 gpt-4o-mini）
- 无 API Key 时使用前端 Mock 数据

### 3.6 测评结果（RESULT）

**Tab 结构：**

| 条件 | 显示的 Tab |
|------|-----------|
| 有 AI 报告 | [测评结果] [深度报告] |
| 无 AI 报告（跳过简历或 API 失败） | [测评结果] |

#### Tab 1：测评结果

**五维人格雷达图：**
- 用户得分 vs 常模参考值双线对比
- 用户得分：绿色实线
- 常模参考：灰色虚线

**雷达图下方分数卡片：**
- 5 列，每列：维度得分 + 维度名称 + 与常模差值（+/-）

**各维度得分详情（进度条）：**
- 每个维度一行：维度名称 + 得分
- 进度条范围 1.0 ~ 5.0，主题色填充
- 常模位置用竖线标记

**维度说明卡片：**
- 5 张卡片，每张含维度名称 + 文字说明

**补充上传入口（仅无报告时显示）：**
- 测评结果底部展示卡片按钮
- 文案："上传简历，获取 AI 深度报告"
- 副文案："结合简历生成职场画像、岗位推荐与成长建议"
- 点击后跳转至简历上传页

#### Tab 2：深度报告（仅有报告时显示）

**核心画像板块：**
- 从报告 Markdown 中提取 `## 核心画像` 段落
- 按 `**粗体标题**` 拆分为子卡片（如核心优势、潜在短板、内在张力）
- 每张子卡片：圆形指示点 + 标题 + 内容

**最佳战场板块：**
- 从报告 Markdown 中提取 `## 最佳战场` 段落
- 按 `**粗体标题**：内容` 格式拆分为子卡片
- 每张子卡片：标题 + 内容
- 板块底部：岗位推荐卡片列表 + "换一换"按钮

**成长引擎板块：**
- 优先使用结构化 JSON 数据（`growthSuggestions`）
- 每条建议一张卡片：序号圆圈 + 标题 + 描述
- 卡片下方：匹配的学习资讯链接
- 若无结构化数据，降级使用报告 Markdown 中 `## 成长引擎` 段落

**寄语板块：**
- 从报告 Markdown 中提取 `## 寄语` 段落
- 引用块样式展示

**免责声明：**
- 底部黄色提示框
- "本报告由 AI 基于性格测评数据与简历内容生成，仅供参考，不构成专业心理咨询或职业规划建议。"

### 3.7 岗位推荐

**数据来源：** 前端岗位库，根据性格分数 + 荣格分数 + MBTI 类型匹配

**展示规则：**
- 每次展示 3 个岗位卡片
- "换一换"按钮刷新推荐
- 每次从匹配池中随机抽取，避免重复
- 已展示过的岗位 ID 记录在 `excludedJobIds` 中
- 当所有匹配岗位都展示过后，重置排除列表重新随机
- 点击岗位卡片跳转至岗位详情页

---

### 3.8 历史记录（HISTORY）

**入口：** 测评主页右上角历史图标

**列表展示：**
- 按时间倒序排列
- 每条记录显示：
  - 时间戳（格式：YYYY-MM-DD HH:mm）
  - 测评类型标签：BFI 测评（绿色）/ 手动输入（紫色）/ 综合测评（琥珀色）
  - 性格标签（如"ENFP · 高外向性"）
  - 报告状态：
    - 有报告 → "已生成报告"
    - 跳过简历 → "仅测评结果" + "未上传简历"灰色标签
  - 右侧："查看详情 →"
  - 右上角：删除按钮

**删除操作：**
- 点击删除按钮弹出确认弹窗
- 标题："删除记录"
- 内容："确定要删除这条测评记录吗？删除后无法恢复。"
- 按钮：[取消] [删除]

**空状态：** 图标 + "暂无测评记录" + "完成一次测评后记录将出现在这里"

**存储限制：** 最多保留最近 20 条记录，超出自动淘汰最早的

---

### 3.9 历史详情（HISTORY_DETAIL）

**页面结构：** 与结果页一致

**Meta 信息：** 时间戳 + 性格标签

**Tab 逻辑：** 同结果页——有报告时显示两个 Tab，无报告时只显示"测评结果"

**补充上传入口：** 无报告的历史记录，测评结果底部展示"上传简历，获取 AI 深度报告"卡片按钮。点击后将该记录的分数数据加载到状态中，跳转至简历上传页。生成报告后保存为新的历史记录。

**岗位推荐：** 从历史记录的分数数据实时匹配，支持"换一换"刷新

## 四、数据模型

### 4.1 历史记录（History Record）

```
{
  id: string,                    // 唯一标识，格式 "rec_{timestamp}"
  timestamp: number,             // 创建时间戳
  assessmentType: string,        // "bfi" | "manual" | "both"
  bfiScores: {                   // BFI 五维度均分（BFI 测评时有值）
    E: number,                   // 1.00 ~ 5.00
    A: number,
    C: number,
    N: number,
    O: number
  } | null,
  mbtiType: string | null,       // 如 "ENFP"（手动输入时有值）
  jungScores: {                  // 荣格八维分数（手动输入时有值）
    Ti: number, Te: number,      // 0 ~ 100
    Fi: number, Fe: number,
    Ni: number, Ne: number,
    Si: number, Se: number
  } | null,
  personalityTag: string,        // 如 "ENFP · 高外向性"
  resumeText: string,            // 简历纯文本（跳过时为空）
  report: string,                // AI 生成的 Markdown 报告（跳过/失败时为空）
  jobTypeRecs: Array,            // AI 岗位类型推荐 JSON
  growthSuggestions: Array,      // AI 成长建议 JSON
  resumeSkipped: boolean         // 是否跳过了简历上传（仅跳过时为 true）
}
```

### 4.2 答题进度（Quiz Progress）

```
{
  shuffledIds: number[],         // 打乱后的题目 ID 序列
  currentIndex: number,          // 当前题目索引（0-based）
  answers: { [questionId]: 1~5 },// 已作答的题目及选项
  timestamp: number              // 最后保存时间
}
```

### 4.3 简历缓存（Resume Cache）

```
{
  text: string,                  // 简历纯文本
  source: string,                // 来源（文件名或 "manual_input"）
  timestamp: number              // 缓存时间
}
```

## 六、状态机流转

```
HOME
  │ 点击"开始测评"
  ▼
WELCOME
  ├─ 点击"开始测评"────────────→ QUIZ
  ├─ 点击"手动输入"────────────→ MANUAL_INPUT
  ├─ 点击"历史记录"────────────→ HISTORY
  └─ 有进度弹窗 → 继续 ───────→ QUIZ 或 RESUME（取决于答题进度）
                 → 重新开始 ──→ 清除进度，留在 WELCOME

QUIZ
  │ 答完第 44 题，自动计算分数
  ▼
RESUME ←── MANUAL_INPUT 提交后也进入此步骤
  ├─ 提交简历 ─→ LOADING → RESULT
  └─ 跳过简历 ─→ RESULT（无报告）

RESULT
  ├─ 点击返回 ──→ WELCOME
  ├─ 点击岗位 ──→ JOB_DETAIL
  └─ 点击"上传简历"（无报告时）──→ RESUME

HISTORY → 点击记录 → HISTORY_DETAIL
  ├─ 点击岗位 ──→ JOB_DETAIL
  └─ 点击"上传简历"（无报告时）──→ RESUME（加载历史分数）
```

---

## 七、历史记录写入时机

| 场景 | 是否写入历史 | 记录内容 |
|------|-------------|---------|
| 提交简历 + AI 生成成功 | 是 | 完整记录（分数 + 简历 + 报告 + 推荐） |
| 提交简历 + AI 生成失败 | 是 | 部分记录（分数 + 简历，报告为空） |
| 跳过简历 | 是 | 部分记录（分数，标记 `resumeSkipped: true`） |
| 答题中途退出 | 否 | 进度保存在 `bfi_quiz_progress`，下次可恢复 |
| 简历步骤退出 | 否 | 进度保存在 `bfi_quiz_progress`，下次可恢复至 RESUME |

## 八、接口需求（后端）

### 8.1 深度报告生成

```
POST /chat/completions
Content-Type: application/json
Authorization: Bearer {API_KEY}

Request:
{
  model: string,
  messages: [
    { role: "system", content: "你是一位精通心理学和职业规划的顶级专家。请用中文回答，使用 Markdown 格式。" },
    { role: "user", content: "{prompt}" }
  ],
  stream: true,
  temperature: 0.8,
  max_tokens: 3000
}

Response: SSE 流式返回 Markdown 文本
```

**Prompt 输入变量：**
- BFI 五维度均分 + 常模参考
- MBTI 类型（可选）
- 荣格八维认知功能分数（可选）
- 简历纯文本

### 8.2 岗位类型推荐

```
POST /chat/completions

Request:
{
  model: string,
  messages: [...],
  temperature: 0.7,
  max_tokens: 500
}

Response: JSON 数组
[{ "type": string, "reason": string, "tags": string[] }]
```

### 8.3 成长建议

```
POST /chat/completions

Request: 同上

Response: JSON 数组
[{ "title": string, "description": string, "tags": string[] }]
```

> 注：当前三个接口均通过 OpenAI Chat Completions 格式调用。如后端需自建网关代理，保持该接口格式即可无缝对接。

| text-secondary | #6b7c72 | 辅助文字 |
| border | #d8e4d4 | 边框、分割线 |

### 设计原则
- 全局统一使用主题绿色（primary），不按维度/板块分配不同颜色
- 卡片统一使用 `glass` 样式（半透明白底 + 模糊背景）
- 子卡片使用 `bg-bg-dark/50 rounded-xl p-4` 样式
- 圆角统一：大卡片 `rounded-3xl`，子卡片 `rounded-xl`，按钮 `rounded-2xl`
- 弹窗使用 `ModalOverlay` 组件，遮罩 + 居中卡片 + 双按钮
